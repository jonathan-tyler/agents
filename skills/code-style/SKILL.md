# Code Style Skill

## Rules

- Apply this comment style across all programming languages unless a project explicitly overrides it.
- Place explanatory comments on the line above the code they describe.
- For non-code prose comments, start with a space after the comment marker and use sentence case.
- Wrap long comments to approximately 100 characters.
- Keep comments focused on intent and context, not obvious restatements of code.
